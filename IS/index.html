Как по делу применить UML для проектирования программ на JavaScript?

Простейший жизненный цикл ПО

  1. Формирование ТЗ (взаимодействие с заказчиком). -> BA/SAn/PM
  2. Проектирование структуры программы. -> SArch
  3. Разработка программы. -> programmers
  4. Тестирование и отладка. -> QA
  5. Внедрение. -> PE
  6. Эксплуатация и сопровождение. -> programmers, PE, SAdm

Количество участников зависит от масштабов проекта, наличие документации - от его цели.

Коротко о главном

  • Unified Modeling Language (UML)
  • Model Driven Architecture (MDA)
  • JavaScript

Преимущества использования UML

  1. Возможность взаимодействия с людьми без технического образования. ;)
  2. Планирование временных затрат.
  3. Ввод некоторых правил и ограничений в проекте.
  4. Рефакторинг кода.
  5. Кодогенерация.
